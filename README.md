# CF-Hunter GUI (Cloudflare 优选 IP 猎手)

这是一个完全静态的、基于 React 18 和 Tailwind CSS 构建的单页应用（SPA）。它的核心目标是帮助网络极客和普通用户，通过简单直观的可视化表单，**毫秒级实时生成**用于 [FOFA](https://fofa.info/) 和 [Censys](https://search.censys.io/) 的复杂反代 IP 搜索语法，免去死记硬背复杂检索语法的烦恼。

## 🌟 核心特性 (技术亮点)

- **双引擎底层驱动**：同时支持 FOFA (标准检索逻辑) 与 Censys (最新 CenQL 2.0 Host Data Model 嵌套规范)。
- **智能中英映射引擎**：输入地区时（如“新西兰，阿根廷”），底层解析器会自动将中文国家名翻译为对应的双字母 ISO 代码或英文全称，无缝对接检索平台。
- **宽容的分隔符支持**：所有自定义输入框均同时支持**中英文全角/半角逗号**及**多重空格**的混合输入。
- **极简大网模式 (Smart Defaults)**：默认采用“403 拦截特征 + 全局不限时 + 不限区域 + 不限厂商”的最宽容匹配策略，保证初始查询的绝对出种率。
- **配置链接毫秒级分享**：所有复杂的表单勾选项和输入内容，会通过 LZString 进行超高压缩率的序列化，并实时倒影在 URL 哈希 `#config=` 中。复制链接即可将整个配置面板的状态完美分享给群友。
- **纯净静态架构**：100% 前端运算，不调用任何第三方接口，极致隐私安全，支持任意静态托管平台零成本部署。

## ✨ 核心功能展示 (你能用它做什么)

1. **可视化表单拼图**：无需背诵 `asn=="13335"` 或 `http.response.status_code=403`，只需动动鼠标勾选“地区”、“厂商”、“端口”，即可在底部**实时预览**生成的复杂长串查询语法。
2. **多模式快捷输入**：支持勾选内置的热门优质 ASN 厂商（如 DMIT、搬瓦工等线路），也支持在文本框里手动批量粘贴你私藏的端口号、黑名单 ASN 等。
3. **高级极客过滤**：提供“只看 IPv4”、“剔除已知建站域名”、“一键剔除 Cloudflare 官方公开 IP 段”等高阶扫盘防污染功能。
4. **一键安全复制**：系统会自动对 FOFA 语法进行安全的 Base64 编码。用户只需点击复制，粘贴到已登录的 FOFA 或 Censys 面板中即可无缝搜索，杜绝编码错误。
5. **历史记录与导出**：自动缓存最近 10 次成功生成的查询配置；支持一键复制原生语法，或将其导出为本地 `.txt` 文本留存备用。

## 🛠 技术栈

- **框架**: React 18, TypeScript, Vite
- **样式**: Tailwind CSS (Dark Mode 完美适配)
- **状态管理**: `useReducer` 聚合状态树
- **图标**: Lucide React
- **辅助依赖**: `lz-string` (配置压缩), `react-hot-toast` (极简弹窗)

## 📦 部署指南

本项目是一个完全静态的纯前端应用（构建输出为纯 HTML/CSS/JS），**无需任何后端服务器**。您可以根据自己的使用场景，选择以下三种部署方式：

### 1. ☁️ Cloudflare Pages (零成本推荐)
如果您希望拥有一个长期公开在线、自带 HTTPS 与全球极速 CDN 的稳定版本，强烈推荐使用 Cloudflare Pages。
1. 将本项目代码完整推送到您的 GitHub 私有或公开仓库。
2. 登录 [Cloudflare 控制台](https://dash.cloudflare.com/)，进入 **Workers 和 Pages** 选项卡。
3. 点击 **创建** -> 选择 **Pages** -> 选择 **连接到 Git**。
4. 授权 GitHub 并选择本项目的仓库。
5. 在**设置构建和部署**页面，填写以下关键信息：
   - **框架预设**：选择 `Vite` (如果没有，选 `None` 也可以)
   - **构建命令**：填写 `npm run build`
   - **构建输出目录**：填写 `dist`
6. 点击 **保存并部署**。几十秒后，Cloudflare 就会为您分配一个可永久免费访问的 `xxx.pages.dev` 域名。

### 2. 🐳 Docker (自建托管)
如果您拥有自己的 VPS 服务器（如独立机器、群晖 NAS 等），并且希望自己掌控一切数据，可以使用 Docker 配合 Nginx 部署。
> **前置要求**：您需要在项目根目录自行创建一个 `Dockerfile`。推荐使用两阶段构建：先用 Node 镜像编译，再把产物丢给 Nginx 镜像。
1. **构建镜像**：在项目根目录执行（会自动执行 npm 构建并打包进 Nginx）
   ```bash
   docker build -t cf-hunter-gui .
   ```
2. **启动容器**：将容器的 80 端口映射到宿主机的 8080 端口（可自行修改）
   ```bash
   docker run -d -p 8080:80 --name cf-hunter --restart always cf-hunter-gui
   ```
3. 访问 `http://您的服务器IP:8080` 即可使用。

### 3. 💻 本地极简开发运行
如果您纯粹只想在自己的个人电脑上偶尔用一下，甚至不想将其公开到互联网上，直接用 Node.js 本地跑是最安全省事的。
> **前置要求**：电脑上已安装 [Node.js](https://nodejs.org/)。
1. 下载源码并进入项目目录，安装必要依赖：
   ```bash
   npm install
   ```
2. 启动本地开发服务器（带有热更新功能）：
   ```bash
   npm run dev
   ```
3. 浏览器会自动打开 `http://localhost:5173`。所有生成的查询语法和压缩配置都会在您的本地浏览器沙盒内完成。
*(注：如果需要输出纯静态的 HTML 网页给其他内网机器用，可以运行 `npm run build`，拿走 `dist/` 文件夹里的东西即可。)*

## 💡 核心设计理念 (关于 ASN 与 优选)

本工具摒弃了传统的“拼接超长 IP 段 CIDR”的笨拙做法，因为这种做法极易导致生成的查询语句突破浏览器 URL 2000 字符的长度限制，引发 `414 URI Too Long` 错误。

取而代之的是，本工具深度挖掘并利用了 BGP 网络的本质——**ASN (自治系统编号)**。当您勾选某个优质服务商（如 DMIT）时，引擎实际上是定位到了该服务商所处的优质网络骨干网（AS906 / AS54574）。不仅能扫出该商家的机器，连同在该网段下的其他超低延迟“邻居”节点也能一并捕获，这才是**真正意义上的“路由优选”**。
